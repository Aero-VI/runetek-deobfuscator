# RuneTek Universal Deobfuscator & Modification Framework

## Overview

A Java-based framework using ASM bytecode manipulation to deobfuscate RuneTek game clients and produce clean, re-compilable Java source projects. The engine uses a pipeline architecture inspired by C# enterprise patterns (dependency injection, strategy pattern, pipeline middleware).

## Technology Stack

- **Language:** Java 21
- **Build:** Maven
- **Bytecode:** ASM 9.7.1 (for reading/writing/transforming .class files)
- **Decompilation:** Procyon 0.6.0 (for final source output)
- **JSON:** Gson 2.11.0 (for mappings and hook definitions)
- **Logging:** SLF4J 2.0.13
- **Testing:** JUnit 5.10.3
- **Patterns:** Pipeline, Strategy, Service Locator/DI

## Phase 1: Heuristic Mapping & Renaming

Analyzes obfuscated classes to identify patterns and rename them meaningfully:

### Structural Analysis (ClassHeuristicAnalyzer)
- **Client** вҖ” extends Applet/Canvas/JFrame with main(String[])
- **Widget** вҖ” 8+ int fields, array fields, 12+ total fields
- **Node** вҖ” 2+ self-referencing fields, вүӨ5 total
- **Cache** вҖ” object array + int size + 3-8 methods
- **Buffer** вҖ” byte[] + int + 5+ methods
- **Model** вҖ” 5+ int[] fields, 10+ total fields
- **Definition** вҖ” String + int fields, 15+ total

### String Literal Analysis (StringLiteralAnalyzer)
Falls back to keyword matching when structural analysis doesn't match:
- Login/password strings вҶ’ LoginHandler
- Socket/connection strings вҶ’ NetworkHandler/Connection
- Render/draw/rasterize strings вҶ’ Rasterizer/GLRenderer
- Cache/archive/idx strings вҶ’ CacheManager/Archive
- Player/NPC/entity strings вҶ’ Player/NPC/Entity
- And many more (audio, chat, inventory, map, etc.)

### Field & Method Naming
- **FieldPatternMatcher** вҖ” context-aware field naming by class type (Widget.x/y/width/height, Buffer.payload/position, etc.)
- **MethodSignatureMatcher** вҖ” signature patterns (getters, setters, read/write), instruction patterns (graphics calls, socket ops, math-heavy), and string constant clues

### Mapping Persistence
- MappingStore supports JSON import/export for reuse across client revisions
- All mappings applied atomically via ASM's Remapper infrastructure

## Phase 2: ASM Bytecode Injection Hooks

### Hook Types
1. **METHOD_ENTRY** вҖ” fires at method start
2. **METHOD_EXIT** вҖ” fires before each return instruction
3. **FIELD_GET** вҖ” fires after each GETFIELD/GETSTATIC (value passed as argument)
4. **FIELD_SET** вҖ” fires before each PUTFIELD/PUTSTATIC (new value passed as argument)

### EventBus (Injected into Target JAR)
Generated as pure bytecode (no source dependency):
- `EventBus.fire(String eventName, Object[] args)` вҖ” dispatches to registered listeners
- `EventBus.register(String eventName, EventListener listener)` вҖ” registers a listener
- `EventBus.unregister(String eventName, EventListener listener)` вҖ” removes a listener
- Thread-safe: ConcurrentHashMap + CopyOnWriteArrayList

### EventListener Interface (Also Injected)
```java
public interface EventListener {
    void onEvent(String eventName, Object[] args);
}
```

### Primitive Boxing
Field hooks automatically box primitive values (intвҶ’Integer, longвҶ’Long, etc.) before passing to the EventBus.

## Output Formats

1. **Class directory** вҖ” individual .class files (always produced)
2. **JAR file** вҖ” runnable JAR with manifest (--output-jar)
3. **Maven project** вҖ” decompiled .java sources via Procyon (--decompile)
4. **Mapping JSON** вҖ” reusable name mappings (--mappings)
